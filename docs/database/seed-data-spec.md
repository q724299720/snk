# 冷启动种子数据规范

## 文档职责

本文件定义 `Phase 0` 冷启动种子数据的字段要求、最低规模、来源优先级和导入前校验规则。

## 种子数据目标

首批数据需要同时支持：

- 文本搜索
- 条形码检索
- OCR 文本召回
- 基础记录创建

## 首批数据来源优先级

1. 自建整理的高频品牌与高频菜品
2. 外部开放食品数据库
3. 人工补充高频 OCR 关键词
4. 后续用户创建的 `pending` 条目

## 建议最小字段集

种子数据至少包含以下字段：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `name` | 是 | 主展示名 |
| `category` | 是 | 一级分类 |
| `subcategory` | 否 | 二级分类 |
| `brand` | 否 | 品牌 |
| `barcode` | 否 | 包装食品优先补齐 |
| `alias` | 否 | 常见简称 |
| `search_keywords` | 是 | 搜索和 OCR 召回关键词 |
| `tags` | 否 | 口味 / 场景 / 功能标签 |
| `source` | 是 | `system / external_api` |
| `audit_status` | 是 | 首批系统数据统一为 `approved` |

## 手动创建 pending 条目最小字段

当搜索、扫码、OCR、图像识别都失败时，用户手动创建 `pending FoodItem` 的最小必填字段为：

- `name`
- `item_type`
- `category`

其他字段如 `brand`、`barcode`、`alias`、`search_keywords`、`tags`、图片均为选填，后续可由审核补全。

其中分类字段规则为：

- 一级分类 `category` 必选
- 二级分类 `subcategory` 选填

图片字段规则为：

- 图片选填，不作为手动创建 `pending FoodItem` 的必需条件
- 若用户提供图片，可用于后续审核、OCR 补全和条目纠错

## search_keywords 规则

`search_keywords` 建议包含：

- 品牌名
- 品类名
- 口味词
- OCR 易识别出的包装文字
- 常见错别字或简称

示例：

```text
乐事,薯片,黄瓜味,青柠味,袋装,休闲零食
```

## 首批数据最低规模

- 总条目数：3000 条以上
- 带条形码条目：1500 条以上
- 每个一级分类至少 50 条
- 每个高优先品牌至少 20 条

## 导入前校验规则

### 必过校验

- `name` 不为空
- `category` 必须属于已定义一级分类
- `source` 必须合法
- `audit_status` 必须合法

### 条形码校验

- 同一条形码不能映射多个不同 `FoodItem`
- 条形码为空的条目必须有足够的 `search_keywords`

### 搜索召回校验

- `name + alias + search_keywords` 拼接后长度不能过短
- 高优先品牌条目必须至少含有品牌词和口味词 / 品类词之一

## 建议文件格式

首批导入建议使用 CSV 或 TSV，方便人工整理和脚本处理。

建议列顺序：

```text
name,category,subcategory,brand,barcode,alias,search_keywords,tags,source,audit_status
```

## Phase 0 Step 1 验收标准

- 已定义最小字段集
- 已定义最低规模
- 已定义导入前校验规则
- 可直接指导后续编写导入脚本或整理 CSV
