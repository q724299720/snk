# 首批食物分类与标签体系

## 文档职责

本文件定义 `Phase 0` 需要优先落地的食物分类、标签体系和冷启动数据覆盖范围，作为种子数据导入的唯一基线。

## 冷启动目标

首批数据不是追求全量，而是追求：

- 搜索可命中
- 扫码可命中
- 常见零食和家常菜有基础覆盖
- 便于后续 OCR 关键词召回

建议首批目标规模：

- `FoodItem`：3000-5000 条
- 覆盖条形码零食 / 饮料：1500 条以上
- 覆盖常见中餐菜品：500 条以上
- 覆盖水果 / 甜品 / 快餐 / 烘焙：500 条以上
- 覆盖高频品牌别名与 OCR 关键词：每条尽量补齐

## 命名规范

- `name` 采用用户最常搜索的标准展示名
- `alias` 保存常见简称、口语名称
- `search_keywords` 保存 OCR 文本碎片、口味词、品牌词、包装词
- 条形码食品必须尽量补齐 `barcode`
