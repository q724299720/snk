# SNK

美食 / 零食分享与记录项目的核心语义定义，避免在商品、菜品、记录对象等概念上混用术语。

## Language

**FoodItem**:
系统中统一的可被搜索、识别、记录和审核的目录实体，既可以表示包装食品商品，也可以表示菜品或其他可食用对象。通过 `item_type` 区分其语义类型，而不是一开始拆成多个目录实体。对于 `packaged_product`，同一条形码视为同一目录实体；对于 `dish`，先按通用菜名建统一条目，不按店铺拆分。
_Avoid_: ProductItem, DishItem, 商品条目, 菜品条目

**FoodRecord**:
用户围绕某个 `FoodItem` 产生的一次具体发生的记录事件，表示一次吃过、买过、试过或评价过的行为，而不是长期汇总卡片。评分、备注、图片、价格、地点等都附着在这次事件上。正式提交到服务端的记录必须绑定一个 `FoodItem`，但客户端本地草稿允许在识别完成前暂时未绑定。`FoodRecord` 默认仅自己可见，公开应当是后续用户主动触发的动作。
_Avoid_: 长期评价卡片, 收藏卡片, 食物档案
